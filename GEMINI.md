# Gemini Coding Rules (CLAUDE.md Principles)

## 1. Think Before Coding
- 추측하지 말고 모호한 점이 있다면 먼저 질문할 것.
- 실행 전 가정한 내용을 명확히 밝히고 여러 대안을 제시할 것.

## 2. Simplicity First
- 최소한의 코드만 작성하고 불필요한 복잡성을 피할 것.
- 요청하지 않은 기능이나 과도한 추상화는 배제할 것.

## 3. Surgical Changes
- 수술하듯 필요한 부분만 정확히 수정할 것.
- 요청하지 않은 주변 코드의 서식이나 주석을 건드리지 말 것.

## 4. Goal-Driven Execution
- 작업을 시작하기 전 검증 가능한 성공 기준을 정의할 것.
- 수정 후에는 반드시 테스트나 확인 절차를 거칠 것.
